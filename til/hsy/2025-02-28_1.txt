3. 오토 인코더와 변분 오토 인코더
1) 오토 인코더의 이해

<오토인코더 (Autoencoder, AE)>

* 개념
- 입력 데이터를 인코더(Encoder)를 통해 압축하고, 디코더(Decoder)를 통해 다시 복원하는 신경망 구조
- 목적: 데이터 차원 축소, 노이즈 제거, 이상치 탐지

* 구조
- 인코더: 입력 데이터를 저차원 잠재 표현(latent space)으로 변환
- 디코더: 잠재 표현을 다시 원본 데이터로 복원
- 손실 함수: 원본 데이터와 복원된 데이터 간의 차이(보통 MSE, Mean Squared Error) 최소화

* 활용
- 특징 추출(Feature Extraction): 인코더 부분을 활용해 차원 축소된 표현을 생성
- 이상치 탐지(Anomaly Detection): 정상 데이터는 잘 복원되지만, 이상치는 큰 오차를 보임


2) 변분 오토 인코더의 이해

<변분 오토인코더 (Variational Autoencoder, VAE)>

* 개념
- AE의 확장 버전으로 데이터를 확률적 분포에서 샘플링하여 보다 다양한 데이터를 생성할 수 있음
- 차이점: AE는 특정 값 하나를 출력하지만 VAE는 평균과 분산을 학습하여 확률 분포에서 샘플링

* 구조
- 확률적 인코더: 입력 데이터에서 평균과 분산을 학습하여 정규분포를 근사
- 잠재 변수 샘플링: z=μ+σ⋅ϵ, 여기서 𝜖∼𝑁(0,1)
- 확률적 디코더: 샘플링된 z를 통해 원본 데이터를 생성

* 손실 함수 (ELBO, Evidence Lower Bound)
- 복원오차 (Reconstruction error): 원본 데이터와 복원된 데이터 간의 차이
- 정규화 항 (Regularization term): 인코더의 출력 분포가 표준 정규 분포에 가깝도록 제약

* 활용
- 새로운 이미지 생성
- 의미있는 잠재 표현을 학습하여 데이터 분류, 클러스터링 등에 사용

3) 벡터 양자화 변분 오토 인코더의 이해

<벡터 양자화 변분 오토인코더 (VQVAE, Vector Quantized Variational Autoencoder)>

* 개념
- VAE의 한계(연속적인 잠재 변수로 인한 흐릿한 생성 결과)를 극복하기 위해 이산(Discrete) 잠재 변수를 도입한 모델
- 잠재 변수를 코드북(Codebook) 내의 고정된 벡터 중 하나로 양자화(Quantization)

* 구조
- 인코더: 입력을 낮은 차원의 연속적인 벡터로 변환
- 양자화: 연속적인 벡터를 코드북의 가장 가까운 벡터로 매핑
- 디코더: 양자화된 벡터를 원본 데이터로 복원

* 특징
- 잠재 변수를 연속적인 벡터가 아닌 이산적인 코드북의 벡터 중 하나로 표현
- PixelCNN을 사용하여 잠재 변수의 공간적인 관계를 학습하여 생성 모델로 활용

<VQVAE-2>
- 고해상도 이미지 생성을 위해 계층적 구조를 도입
- 위층(Top Level): 전역적 특징(Global Features) 학습
- 아래층(Bottom Level): 세부적 특징(Local Details) 학습
